# HTTP Server with ThreadPool & Handlers

Многопоточный HTTP-сервер на Java без фреймворков. Проект демонстрирует подход к архитектуре приложения: разделение ответственности между классами, безопасная работа с многопоточностью, парсинг бинарных протоколов.

## Архитектура

### Server.java — главный класс

**Что он делает:**
- Принимает входящие соединения на ServerSocket
- Распределяет каждое соединение в пул из 64 потоков
- Парсит HTTP-запросы
- Находит и вызывает нужный обработчик
- Отправляет ответ клиенту

**Парсинг HTTP:**

String requestLine = reader.readLine();
String[] parts = requestLine.split(" ");
String method = parts; 
String path = parts;
